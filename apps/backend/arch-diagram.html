Net effect: Merchant installs via Ratio App Store OAuth → Adapter stores merchant in MySQL → Merchant registers via admin form → Adapter calls RP's /shopify-webhook/v1/os-install → RP creates StoreDetail. OS fires order webhooks → Adapter normalizes (paise→rupee, string→numeric ID) → upserts directly into RP's MongoDB shopifyorders collection. Customer submits return via RP portal (or SDK) → RP BE reads from local MongoDB or adapter → Merchant approves → Adapter calls OS Order Service refund APIs. RP BE uses OsAxios as a drop-in ShopifyAxios replacement for ALL data access.
