# Learnings (cross-cutting gotchas)

Non-obvious facts discovered while building. Newest first. Add via the `remember`
skill. Keep each entry 1–3 lines.

- **2026-07-15** — The Ratio dashboard iframes the app admin **in the viewer's browser**, so a `localhost` ADMIN_BASE_URL works single-machine (the delhivery dev pattern) — but platform→app **webhooks come server-side** and never reach localhost. Publishing for other people needs public URLs; with one static ngrok domain, `SERVE_FORMS_ADMIN_DIST` (opt-in, main.ts) mounts the built admin at `/admin-forms/` so one origin serves API + admin (hash routing ⇒ no SPA rewrites).
- **2026-07-15** — ngrok **free tier intercepts EVERY browser-UA request** (even `Accept: application/json` fetches) with its warning page unless `ngrok-skip-browser-warning` is sent. forms-sdk sends it only for `*.ngrok-free.*` API bases; the public-CORS allowlist must include the header or preflight fails. Plain `<script src>` can't carry it — demo pages fetch+inject. Vanishes on real hosting.
- **2026-07-15** — Nest registers no OPTIONS routes, so CORS **preflight to `/public/v1/*` 404s** and `enableCors` (origin-checker + credentials) can't do wildcard for shopper-facing endpoints. Fastify `onRequest` (answer OPTIONS 204) + `onSend` (stamp `ACAO:*`, strip allow-credentials) hooks scoped to `PUBLIC_SUBMIT_RE` in main.ts is the pattern. Sandbox ecosystem base told by platform team: `https://api-gw-v4.dev.gokwik.in/sandbox/aes`.
- **2026-06-26** — Wizzy's search/autocomplete (`api.wizsearch.in/v1`) are **public** endpoints: auth = `x-store-id` + a **public** `x-api-key` only, CORS wildcard, and the store **secret** is explicitly forbidden on them (only the catalog write endpoints need it). So a storefront SDK can call Wizzy **directly from the browser** — no backend proxy needed. The public-config endpoint must never return the secret.
- **2026-06-26** — For an embeddable SDK with a hard size budget (`packages/wizzy-sdk`, ≤10KB widget), import shared Zod schemas as **type-only** (`import type …`) so Zod is erased at build and never bundled; do runtime validation on the backend. Also ship the overlay and the faceted results page as **separate** ESM bundles (the loader injects the heavy results bundle only on the results route) to keep the always-loaded overlay small.
- **2026-06-26** — Playwright E2E sharing one static server + the same `page.route` mock patterns across specs flakes under parallel workers (intermittent navigation timeouts). Run it serial: `fullyParallel:false, workers:1, retries:1`.
- **2026-06-22** — Ratio product/variant prices are **integer PAISE (minor units)** — divide by 100 for GMC. Verified against a **live `products/update` webhook** (an OSMO combo: `price:155900` / `compare_at_price:209800` = ₹1,559 / ₹2,098; ₹155,900 would be absurd). This **re-confirms the original 2026-06-08 paise finding** and **reverses the 2026-06-18 "rupees" note**, which was based on *synthetic* test fixtures (`29.99`, `199.99`), not real data. `paiseToMajor` applies in BOTH `parseWebhookProduct` and `parseRestProduct`. ⚠️ The Meta catalog transformer was NOT audited for this — check before trusting Meta prices.
- **2026-06-18** — ~~Ratio product/variant prices are rupees (major units), NOT paise~~ _(REVERSED 2026-06-22 — this was off synthetic fixtures; live data is paise. See above.)_
- **2026-06-18** — Real webhook delivery envelope is `{ event_type, merchant_id, product }` — there is **no top-level delivery id or timestamp** (resolves google TRD R1). Core dedupes by deriving `<event_type>:<product.id>` into `webhook_log.ratio_webhook_id`, retry-windowed on `received_at` (~3h) and resting on idempotent handlers — a permanent unique key would wrongly skip a legitimate second update to the same product. Signature header is `x-openstore-signature: sha256=<hex>`.
- **2026-06-18** — The platform exposes products in **two different field shapes**: REST `GET /products` is camelCase (`name`, `description`, `productType`, `inventory.quantity`, `images[].src`, `variants[].sku`) wrapped in `{ success, data[], pagination }`; the webhook `product` is snake_case (`title`, `body_html`, `product_type`, `warehouseQt[]`, `images[].url`, `variants[].variant_id`/`sku_id`, `option1/2/3`). Needs separate normalizers (`parseRestProduct` / `parseWebhookProduct`).
- **2026-06-17** — `buildDefaultEventMap(vendor?)` in `packages/shared/src/schemas/event-map.ts` has explicit branches for `'meta'` (Title-Case CAPI names) and `'moengage'` (Title-Case MoEngage names); posthog and google fall through to the snake_case template default. New vendors that use snake_case defaults need NO new branch — only add a branch if the vendor's canonical event names differ from snake_case.
- **2026-06-17** — `vendor-scaffolder` Step 6 (shared events/config in `packages/shared/`) is NOT optional even for a throwaway dry-run slug: the backend bootstrap file imports `DEFAULT_<SLUG>_HOST` from `@ratio-app/shared/constants/<slug>-events`, so the shared file must exist before `tsc --noEmit` succeeds. The recipe documents this correctly; scaffolders must complete Step 6 before running typecheck.
- **2026-06-17** — Google ID auto-discovery needs only OAuth (no extra scopes/keys): GA4 Measurement IDs come from the GA4 Admin API (`analyticsadmin.googleapis.com/v1beta` → `accountSummaries` → `{property}/dataStreams`, keep `type === 'WEB_DATA_STREAM'` → `webStreamData.measurementId`); GMC account IDs come from the Content API `accounts/authinfo` (`accountIdentifiers[].merchantId`, no merchantId in the path). Ads Conversion ID/label is NOT discoverable by OAuth alone — the Google Ads API requires a developer token (separate Google approval), so Ads stays manual.
- **2026-06-11** — Hosted app pixels register as a SINGLE `<slug>-ratio` pixel (e.g. `google-ratio`, matching `posthog-ratio`), NOT under generic names like `ga4`/`google-ads` (those collide with the storefront's legacy `pixelConfig` keys). Storefront install = add `<Script src=".../<slug>/sdk/<merchantId>.js" strategy="afterInteractive" />` to `layout.tsx` + `"<slug>-ratio": {}` in `src/config/pixelConfig.ts`; per-merchant config is served in the bundle's prelude (not env/pixelConfig).

- **2026-06-08** — Webhook `event` strings use **slash-form** (`products/create`, `app/uninstalled`). _(2026-06-18: confirmed live — the field is `event_type`, slash-form, in a `{ event_type, merchant_id, product }` envelope.)_
- **2026-06-08** — The Web Pixels API (`POST /pixels`) is **Draft** (`codegen_ready:false`); pixel registration must degrade to a `pending_api` status, with script-tag delivery as the working fallback.
- **2026-06-08** — `nest start` runs with cwd = `apps/backend`, and `main.ts` does `dotenv/config` against `DOTENV_CONFIG_PATH='.env'` (cwd-relative); a symlink `apps/backend/.env → ../../.env` makes it load the root `.env`.
